Phase 1 — Flask Core
LAB 01
Routing, URL Rules & Dynamic Segments
Build the skeleton of your app — understand how Flask maps URLs to functions
Beginner
What You'll Learn
  • Create a Flask app instance and run the dev server
  • Use @app.route() with static and dynamic paths
  • Return HTML strings, redirect(), and url_for()
  • Use HTTP method constraints (methods=['GET','POST'])
  • Render the starter template via render_template()
app.py from flask import Flask, render_template, redirect, url_for app = Flask(__name__) @app.route('/') def home(): return render_template('base.html') # ← your code will go here if __name__ == '__main__': app.run(debug=True)
📋 Tasks
  1. Add a route /about that renders a page saying "About Us" using the base template's {% block content %}
  2. Add a dynamic route /user/<username> that greets the user by name — display it in a Bootstrap card inside the content block
  3. Add a route /user/<int:id> — use type converters and return a 404 if id > 1000 using abort(404)
  4. Create a route /go-home that uses redirect(url_for('home')) to redirect back to /
  5. Add navigation links in the navbar by overriding the {% block nav_links %} block in your child templates
💡 Hint
Child templates start with {% extends 'base.html' %} — then override blocks. Flask's url_for('function_name') is safer than hardcoding paths.

LAB 03
Forms, POST Requests & Input Validation
Accept user input, validate it server-side, and respond meaningfully
Intermediate
What You'll Learn
  • Use request.form, request.args, and request.method
  • Build a POST handler that reads and validates form data
  • Use WTForms or manual validation with error messages
  • Use the PRG pattern (Post/Redirect/Get) to avoid double-submits
  • Handle file uploads with request.files
app.py — POST handler pattern @app.route('/contact', methods=['GET', 'POST']) def contact(): if request.method == 'POST': name = request.form.get('name', '').strip() email = request.form.get('email', '').strip() if not name or not email: flash('All fields required', 'danger') return redirect(url_for('contact')) # process... flash(f'Thanks {name}!', 'success') return redirect(url_for('contact')) # PRG return render_template('contact.html')
📋 Tasks
  1. Build a contact form (name, email, message) using the base template — render it with Bootstrap's form classes
  2. Validate all fields on POST — flash error messages for any blank field and re-render the form
  3. Validate that the email contains "@" — flash a specific error if it doesn't
  4. On valid submit, store the submission in a Python list and redirect to a /submissions page that lists all messages
  5. Add a search bar at /search?q=term that reads from request.args and filters the submissions list

LAB 04
SQLite Database with Flask-SQLAlchemy
Persist data properly — models, migrations, CRUD operations
Intermediate
What You'll Learn
  • Set up Flask-SQLAlchemy and configure the DB URI
  • Define models with relationships (one-to-many)
  • Perform create, read, update, delete with SQLAlchemy ORM
  • Use db.session, Model.query.filter_by(), .first_or_404()
  • Run Flask shell to inspect the database
models.py from flask_sqlalchemy import SQLAlchemy from datetime import datetime db = SQLAlchemy() class Post(db.Model): id = db.Column(db.Integer, primary_key=True) title = db.Column(db.String(120), nullable=False) body = db.Column(db.Text, nullable=False) created = db.Column(db.DateTime, default=datetime.utcnow) comments = db.relationship('Comment', backref='post', lazy=True) class Comment(db.Model): id = db.Column(db.Integer, primary_key=True) body = db.Column(db.Text, nullable=False) post_id = db.Column(db.Integer, db.ForeignKey('post.id'), nullable=False)
📋 Tasks

  3. Build a /posts/new form that creates a new Post via POST and saves it to the DB
  4. Build a /posts/<id>/edit route that loads the existing post and updates it on submit
  5. Add a delete button with a /posts/<id>/delete route — use a POST form (not GET!) to delete records safely

LAB 05
Build a REST API with Flask
Turn your Flask app into a JSON API — consumed by your own frontend
Intermediate
What You'll Learn
  • Return JSON with jsonify() and set HTTP status codes
  • Parse JSON request bodies with request.get_json()
  • Design RESTful routes: GET, POST, PUT, DELETE
  • Handle errors with @app.errorhandler returning JSON
  • Call your own API from the HTML page using fetch()
REST API pattern @app.route('/api/posts', methods=['GET']) def api_get_posts(): posts = Post.query.all() return jsonify([{ 'id': p.id, 'title': p.title, 'body': p.body } for p in posts]), 200 @app.route('/api/posts', methods=['POST']) def api_create_post(): data = request.get_json() if not data or 'title' not in data: return jsonify({'error': 'title required'}), 400 post = Post(title=data['title'], body=data.get('body', '')) db.session.add(post) db.session.commit() return jsonify({'id': post.id}), 201
📋 Tasks
  1. Build a full REST API for Post: GET /api/posts, POST /api/posts, GET /api/posts/<id>, PUT /api/posts/<id>, DELETE /api/posts/<id>
  2. Add a JSON error handler for 404 and 400 errors that returns {"error": "..."}
  3. Add a dedicated page in your HTML template with a fetch() call that loads posts from /api/posts and renders them dynamically without page reload
  4. Add a form on the same page that submits a new post via fetch() POST with Content-Type: application/json
  5. Add a simple API key check using a request header (X-API-Key) — return 401 if missing or wrong
LAB 06
Consuming External APIs with requests
Connect your Flask app to the real world — weather, news, data services
Intermediate
What You'll Learn
  • Use the requests library to call external REST APIs
  • Handle API errors, timeouts, and bad responses gracefully
  • Store API keys securely in environment variables
  • Render external API data in your Jinja2 templates
  • Cache API responses to avoid rate limiting
External API call pattern import requests import os @app.route('/weather') def weather(): city = request.args.get('city', 'London') api_key = os.getenv('OPENWEATHER_API_KEY') url = f'https://api.openweathermap.org/data/2.5/weather' try: resp = requests.get(url, params={ 'q': city, 'appid': api_key, 'units': 'metric' }, timeout=5) resp.raise_for_status() data = resp.json() except requests.RequestException as e: flash(f'API error: {e}', 'danger') data = None return render_template('weather.html', weather=data, city=city)
📋 Tasks — pick 2 of 3
  1. Weather App: Use OpenWeatherMap (free tier). Build a search form on the base template page — display temp, description, and icon when city is found
  2. News Feed: Use NewsAPI (free tier). Display top headlines in Bootstrap cards with title, source, and "Read More" link
  3. GitHub Profile: Use https://api.github.com/users/<username> (no key needed). Show avatar, bio, repo count in a profile card
  4. For whichever you build: add a timeout of 5 seconds, catch requests.RequestException, and flash an error message on failure
  5. Store your API key in a .env file and load it with python-dotenv — never hardcode it

FINAL

🔥 Final Challenge — Build & Deploy a Full App

This is your capstone. You'll build a complete deployable web app that combines everything from all 10 labs. No starter code — just the requirements below and your skills.

Option A — "LinkVault" · Bookmark Manager
Users register, save bookmarks (URL + title + tags), organize them, and get AI-suggested tags via an external API call.
  • Full auth: register, login, logout, protected routes
  • CRUD for bookmarks stored in PostgreSQL (or SQLite locally)
  • REST API endpoint: GET /api/bookmarks returning JSON
  • Call the Open Graph API (or use requests + BeautifulSoup) to auto-fill title from URL
  • Tag filtering on the frontend using fetch() with query params
  • Deploy to Render with env vars set
Option B — "PulseBoard" · Personal Dashboard
A dashboard that aggregates live data from 2+ external APIs and displays it in a clean, auto-refreshing single page.
  • Fetch from at least 2 external APIs (weather + news, or GitHub + quotes)
  • Backend caches API responses for 5 minutes using Flask-Caching or a simple dict cache
  • Frontend fetches from your own /api/dashboard endpoint and updates the page every 60s via setInterval + fetch
  • User can customize which city / topics they see (stored in the session or DB)
  • Auth: login required to view the dashboard
  • Deploy to Render — all API keys in env vars
Option C — "FlaskBoard" · Kanban Task Manager
A full task management app with boards, columns, and cards — drag to reorder via a PATCH API call.
  • Models: User → Board → Column → Task (nested relationships)
  • Full REST API: PATCH /api/tasks/<id> to update column (drag & drop)
  • Frontend: fetch() calls for all task moves — no page reload
  • File attachment: upload an image to a task (stored in /static/uploads)
  • Admin blueprint: view all users and boards (admin-only route)
  • Deploy with gunicorn on Render
🏁 Success Criteria
Your app must: (1) run on a live public URL, (2) have at least one authenticated route, (3) call at least one external API, (4) have at least one REST endpoint returning JSON, (5) use environment variables for all secrets.
